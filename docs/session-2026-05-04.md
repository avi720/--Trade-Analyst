### 2. לוגיקת נקודת הסנכרון — מבוסס סטטוס במקום זמן
הנקודה ב-`SyncIndicator` עברה ממדידת "כמה זמן עבר" ללוגיקת סטטוס + קוד שגיאה.

| צבע | מצב |
|---|---|
| ⚫ אפור | אף פעם לא סונכרן |
| 🟢 ירוק | הסנכרון האחרון הצליח (`SUCCESS`) |
| 🟡 כתום | שגיאה transient — הקרון הבא כנראה יתקן |
| 🔴 אדום | שגיאה fatal — דורש פעולת משתמש |

**קודי שגיאה transient (כתום):** 1001, 1004, 1005, 1006, 1007, 1008, 1009, 1017, 1018, 1019, 1021

**קודי שגיאה fatal (אדום):** 1003, 1010, 1011, 1012, 1013, 1014, 1015, 1016, 1020

מקור: [IBKR Flex Error Codes](https://www.ibkrguides.com/clientportal/performanceandstatements/flex3error.htm)

**גם תוקן:** `+ "Z"` ב-`formatShort` וב-`ibkrTitle` כדי שהזמן יוצג נכון (UTC→local).
